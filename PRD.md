# 回家不 - 产品需求文档 (PRD)

## 1. 产品概述

**产品名称：** 回家不  
**一句话描述：** 一个家庭成员"今晚回不回家"的状态共享应用，默认回家，一键标记不回家。  
**目标用户：** 家庭成员（2-10人的小型群组）  
**核心价值：** 做饭的人再也不用挨个问"今晚回来吃饭吗"。

---

## 2. 核心设计理念

### 2.1 极简原则

- **零操作 = 回家**：不打开 App，就代表今晚会回家。状态每天凌晨自动重置。
- **一步操作 = 不回家**：打开 App，点一下，搞定。
- **信息一目了然**：首页即全貌，无需翻页，一眼看清所有人的状态。

### 2.2 设计哲学

> 最好的家庭 App 是你几乎不需要打开的 App。

---

## 3. 用户角色

| 角色 | 说明 |
|------|------|
| **家庭创建者** | 创建家庭群组，生成邀请链接/码，管理成员 |
| **家庭成员** | 加入家庭群组，更新自己的回家状态 |
| **AI Agent** | 通过 API 代用户更新回家状态（如 Siri 捷径、GPT Action 等） |

---

## 4. 功能需求

### 4.1 家庭管理

#### F1: 创建家庭

- 用户注册后可创建一个家庭
- 设置家庭名称（如"老祁家"）
- 自动生成 6 位邀请码 + 邀请链接

#### F2: 加入家庭

- 通过邀请码或邀请链接加入
- 一个用户可以加入多个家庭（如"老齐家" + "合租室友"）

#### F3: 成员管理

- 家庭创建者可移除成员
- 成员可主动退出家庭

### 4.2 回家状态

#### F4: 状态模型

```
状态只有两种：
  ✅ 回家（默认值）
  ❌ 不回家
```

- 每天凌晨 4:00 自动重置所有人状态为"回家"
- 选择凌晨 4 点而非 0 点，是因为有些人会很晚才标记不回家

#### F5: 更新自己的状态

- 首页显示自己的状态卡片
- 点击即可在"回家 / 不回家"之间切换
- 切换时可选填原因（如"加班"、"出差"），也可以不填

#### F6: 查看所有人状态

- 首页以卡片/列表展示所有家庭成员的今日状态
- 显示内容：
  - 头像 + 昵称
  - 状态（回家 / 不回家）
  - 原因（如果填了）
  - 最后更新时间
- 状态排序：不回家的人排在下方/用不同颜色区分

#### F7: 状态变更通知（可选开启）

- 当有人将状态改为"不回家"时，向其他家庭成员推送通知
- 用户可在设置中关闭此通知

### 4.3 用户系统

#### F8: 注册/登录

- MVP 阶段：用户名 + 密码登录
- 后续支持：手机号 + 验证码、Apple ID / 微信登录
- 首次登录设置昵称

#### F9: 个人设置（✅ 已实现）

- 修改昵称（设置页内联编辑）
- 通知偏好（V2）
- 生成/刷新个人 API Token

### 4.4 API 接口（供 AI 调用）

#### F10: RESTful API

为每个用户提供个人 API Token，AI Agent 可通过以下接口操作：

**认证方式：** Bearer Token（个人 API Token）

**接口列表：**

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/v1/auth/register` | 注册新用户 |
| `POST` | `/api/v1/auth/login` | 登录 |
| `GET` | `/api/v1/auth/me` | 获取当前用户信息 |
| `PATCH` | `/api/v1/auth/me` | 修改昵称等个人信息 |
| `POST` | `/api/v1/auth/token/regenerate` | 重新生成 API Token |
| `POST` | `/api/v1/auth/admin/reset-password` | 管理员重置用户密码（需 X-Admin-Key） |
| `GET` | `/api/v1/family` | 获取我的家庭列表 |
| `POST` | `/api/v1/family` | 创建家庭 |
| `POST` | `/api/v1/family/join` | 通过邀请码加入家庭 |
| `GET` | `/api/v1/family/{id}` | 获取家庭详情及成员 |
| `DELETE` | `/api/v1/family/{id}/members/{uid}` | 移除成员或退出家庭 |
| `GET` | `/api/v1/status` | 获取所有家庭今日状态 |
| `GET` | `/api/v1/status/me` | 获取自己的当前状态 |
| `PUT` | `/api/v1/status/me` | 更新自己的回家状态 |
| `GET` | `/api/v1/status/family/{id}` | 获取指定家庭的今日状态 |

**更新状态请求示例：**

```json
PUT /api/v1/status/me
Authorization: Bearer <token>
Content-Type: application/json

{
  "going_home": false,
  "reason": "加班"
}
```

**查询状态响应示例：**

```json
GET /api/v1/status
Authorization: Bearer <token>

{
  "family_name": "老齐家",
  "date": "2026-03-12",
  "reset_at": "04:00",
  "members": [
    {
      "name": "老齐",
      "avatar": "https://...",
      "going_home": true,
      "reason": null,
      "updated_at": null
    },
    {
      "name": "小齐",
      "avatar": "https://...",
      "going_home": false,
      "reason": "加班",
      "updated_at": "2026-03-12T14:30:00+08:00"
    }
  ],
  "summary": "今晚 1 人不回家"
}
```

#### F11: AI 集成场景

提供标准 OpenAPI/Swagger 文档，方便接入：

- **Siri 捷径**："Hey Siri，今晚不回家" → 调用 API 更新状态
- **ChatGPT / GPTs Action**：对话中告诉 AI "今晚不回家了"，AI 自动调用
- **钉钉/飞书机器人**：群内 @机器人 "今晚不回家"
- **Home Assistant**：智能家居联动

## 6. 技术架构建议

### 6.1 前端

| 方案 | 说明 |
|------|------|
| **推荐：Flutter** | 一套代码，iOS + Android + Web |
| 备选：React Native | 社区生态好，但 Web 端支持弱 |
| 备选：小程序 | 微信生态内免安装，但无法做 Push 通知 |

### 6.2 后端

| 组件 | 推荐技术 |
|------|---------|
| **语言/框架** | Python (FastAPI) 或 Go (Gin) |
| **数据库** | PostgreSQL |
| **缓存** | Redis（存当日状态，快速读取） |
| **定时任务** | Celery Beat / Go Cron（每日凌晨重置） |
| **推送通知** | Firebase Cloud Messaging (FCM) / APNs |
| **部署** | Docker + 云服务器 / Serverless |

### 6.3 数据模型

```
User
├── id (UUID)
├── phone / oauth_id
├── nickname
├── avatar_url
├── api_token (用于 AI 调用)
├── created_at

Family
├── id (UUID)
├── name
├── invite_code (6位，定期可刷新)
├── created_by (User.id)
├── created_at

FamilyMember
├── family_id (FK → Family)
├── user_id (FK → User)
├── role (creator / member)
├── joined_at

DailyStatus
├── id (UUID)
├── family_id (FK → Family)
├── user_id (FK → User)
├── date (DATE)
├── going_home (BOOLEAN, default=true)
├── reason (VARCHAR, nullable)
├── updated_at (TIMESTAMP)
├── source (app / api)  ← 区分是手动还是 AI 更新的
```

### 6.4 API 认证流程

```
用户在 App 设置页 → 生成 API Token
         ↓
将 Token 配置到 AI Agent（如 GPTs Action）
         ↓
AI Agent 发请求时携带 Bearer Token
         ↓
后端校验 Token → 识别用户 → 执行操作
```

## 7. 核心业务规则

| 规则 | 说明 |
|------|------|
| **默认回家** | 未做任何操作 = 今晚回家 |
| **每日重置** | 凌晨 4:00 自动重置所有状态为"回家" |
| **只改自己** | 任何人只能修改自己的状态，不能替别人操作 |
| **当日有效** | 状态只代表"今天"，无法提前设置明天 |
| **操作幂等** | 重复设置"不回家"不会产生副作用 |
| **API 等价** | 通过 API 修改状态与在 App 中操作完全等价 |

| 功能 | 说明 |
|------|------|
| **预计到家时间** | 状态为"回家"时可选填预计到家时间 |

| **多日设置** | 支持提前标记未来几天的状态（如出差一周） |
| **智能提醒** | 到了下午 5 点如果还没人标记，推送提醒"今晚回家吗？" |
| **位置联动** | 基于 GPS 自动判断是否在回家路上 |
| **回家倒计时** | 标记"在路上"并共享预计到达时间 |
| **语音更新** | 通过 Siri / 语音助手直接更新状态 |
